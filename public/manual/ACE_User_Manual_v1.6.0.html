ACE — User Guide

Version: v1.6.0 · Updated: 2026-05-10

ACE (Agentic Cue Experience) is an AI-powered live presentation app for worship, conferences, lectures, and theater. It listens to what’s happening in the room and pushes the right slide automatically — no manual clicking through cues during the service.

Tagline: “Ace every cue.”

This guide is the comprehensive reference. For a 60-second walkthrough, use the in-app tour (runs once after first launch; re-trigger from Settings → General → “Show tour again”). For a quick visual map of every function, jump to §17 Feature Index (every panel + button) and §18 Complete Settings Reference.

1. Quick start

  1. Install — drag ACE.app from the DMG into /Applications. First launch may take 10–30 seconds while the bundled AI model warms up.
  2. Animated welcome splash (v1.6) — plays for ~15 seconds on first launch, or click anywhere to skip. Honours prefers-reduced-motion if you have macOS reduce-motion on.
  3. Onboarding wizard (5 steps) — Welcome → Backend ready → Speech model download → macOS permissions (mic + screen-recording) → Done. If a ProPresenter library is detected on your Mac, the wizard auto-prompts the ProPresenter Migrator (see §3) before handing off to the dashboard.
  4. Click Start (top-left of the toolbar) — ACE begins listening.
  5. Sing or speak something the system knows. If you have a song library imported, ACE auto-pushes the matched lyrics. If not, see §3 (Importing content).
  6. To send to a real audience screen, configure outputs via ⌘⇧S (Screen Configuration) — see §5.

Toolbar — left side (detection controls)

  • Start / Stop — toggles AI listening.
  • Mic-level visualizer — traffic-light:
    • Amber + slow pulse → too quiet, speak louder or move closer
    • Green → good signal
    • Red → peaking, pull back or reduce input gain
  • Auto / Song / Bible — detection mode. Auto detects either; Song-only locks to lyrics; Bible-only locks to scripture.
  • Live / Hold — when Live, detected matches go straight to PROGRAM. Hold pauses auto-display (useful between songs).
  • Confidence ± — tweak the AI threshold during a service.

  • AudioVisualizer — see Mic-level above.

Sidebar (left panel)

Library of imported content: songs, presentations (slide decks), Bibles. Drag-reorder to set the Service Plan (the day’s order — songs and scripture in sequence).

Center column

  • Show mode — current and next song lyric sections, scripture passages, slide thumbnails.
  • Edit mode — full lyric/section editor.
  • Bible mode — verse-context view.

Right panel

  • Preview / Program — what’s about to go live (Preview) and what is live (Program). Operators drive most of the service from here.
  • Macros — operator-fired automations (clear all, switch look, etc.).
  • Sermon Notes — sermon recording with auto-detected scripture references.

Media Bin (bottom dock)

Playlists, asset library, single-click trigger, layer/fit, Stock Bin filter (Pixabay if configured). Drag a media item onto the slide preview to make it the audience background.

3. Importing content

⌘I or click Import in the toolbar. Six-stage wizard:

Stage What happens
Source Drag-drop or pick files. Multi-format auto-detect.
Preview Per-item preview (lyrics, scripture, slide titles). Bible references in PPTX titles + speaker notes are auto-detected and tagged per slide.
Resolve Conflict detection — Skip / Replace / Continue per item.
Template (v2) Apply a default theme to imported items.
Import Runs ingestion.
Summary Result counts + per-item status; reference grouping for Bible-detected items.

Supported formats

Type Formats
Songs Plain text, ChordPro / .cho, OpenLyrics XML, OpenSong XML, ProPresenter .pro6 / .pro7 / .pro (v1.6 — newer PP7+ filename), OpenLP .osz (9 formats)
Bibles MyBible .SQLite3, OSIS XML, Zefania XML, OpenSong XML (4 formats)
Slides PowerPoint .pptx, Keynote .key, PDF, image-sequence (numbered PNG / JPEG / WebP / HEIC frames or a folder) (4 sources)

LibreOffice deferred-install

First PPTX import on a Mac without LibreOffice triggers an in-app installer with SHA256-verified mirror download from dl.ace-presenter.app/libreoffice/26.2.3/. ~280 MB per arch. Approve once; future imports use the bundled install.

Migrating from ProPresenter (v1.6)

If you’re switching from ProPresenter, the dedicated ProPresenter Migrator brings your library across in one click. The onboarding wizard auto-prompts it on first launch when it detects a PP install; you can also reach it any time via Settings → Import → “Import from ProPresenter”.

Five-stage wizard:

Stage What happens
Detect Looks for ProPresenter 6 + 7 installs. Reads default + custom library paths from PP’s plist; you can also point it at a path manually.
Preview Scans the library and shows exactly what will come over: song count, media-file count, total size. Nothing is imported yet.
Running Imports songs (with section structure preserved), media (deduped by SHA-256 — no double copies), and playlists.
Done Summary with one-click Undo this import — every record is tagged with an import_session_id, so a single click rolls back the entire run.
Error Surfaces the exact failure (path missing, parse error) with a Retry button.

What’s preserved

  • Songs from .pro7 and .pro (newer PP7+ filename) protobuf files
  • Lyrics extracted from RTF blobs with multi-pass cleanup (preamble groups, residue lines, font-table noise) and de-duplicated by substring containment so the same slide doesn’t appear three times under different protobuf nesting levels
  • Section structure (Verse, Chorus, Bridge, etc.) — one section per ProPresenter slide
  • Media files (images, videos), SHA-256 deduped against your existing Media Bin
  • Playlists, with their original ordering

Storage mode

By default, media files are symlinked from your ProPresenter library path (no disk-space cost; the file lives where PP put it). If you’d rather copy them so the import is self-contained, choose Copy in Preview before running.

Coming in v1.7

Themes / looks mapping — your ACE themes currently apply to imported songs; v1.7 will read PP’s slide styling and map it.

4. Themes and Looks

ACE separates how things look (theme — fonts, colors, background) from what’s enabled (look — which layers render, which themes apply per screen).

Themes (Theme tab)

A theme defines: font family / size / weight / color, subtitle color, background mode (solid / gradient / image), text shadow, position, margins, line-reveal speed.

  • Click Theme in the toolbar.
  • The grid view shows your themes. Click a tile to open the rich editor. Hover ▶ to apply a theme to PROGRAM without editing. Hover ✕ to delete.
  • + New Theme creates a custom one from a default template.
  • Save → the theme persists in your library and can be applied or referenced by Looks.

Looks (Looks tab) — headline v1.4 feature

A Look is a named configuration of: - Which layers render (slide, media, messages, announcements, bible, props) - Which theme each output uses (theme_overrides[screenId] = themeId) - Per-screen layer overrides (one screen shows media, another doesn’t) - Transition duration

Click Looks in the toolbar (between Theme and Stage). Per-Look grid:

                   Default   Audience-1   Audience-2   Stage
Slide              ✓         ✓            ✓            (auto: parent)
Media              ✓         ✓            (off)        ✓
Bible              ✓         ✓            ✓            (off)
Announcements      ✓         ✓            ✓            (off)
Theme override     —         Dark         Scripture    Default

Click Make Live to activate the Look. PROGRAM repaints within ~1 second.

Unified render pipeline (the v1.4 opt-in)

Settings → Screens & Display → Advanced → Unified render pipeline toggle (default off during burn-in).

When ON: the active Look drives every output. Edit a Look → audience repaints in real time. Per-screen overrides take effect.

When OFF: v1.3-style global theme behaviour. Looks store data but don’t drive output.

After one full successful service with the flag on, leave it on. v2 flips the default.

5. Outputs (Screen Configuration)

⌘⇧S opens Screen Configuration — the single source of truth for outputs (replaces the v1.0–v1.2 dashboard NDI/Audience/Stage toggles).

Audience vs Stage

  • Audience = what your congregation sees. Lyrics, backgrounds, scripture. Edge-to-edge.
  • Stage = what the worship team / preacher sees. Confidence layout (current/next slide, timer, clock, sermon notes). Operator-aware chrome.

Multi-display

Both roles support multiple displays. 7 audience screens + 3 stage screens at once if your rig has the displays. Each role’s toggle drives all assigned displays in lockstep.

Display Mode toggle (per output)

  • Borderless Fill (default) — preserves your operator menu bar, multi-display friendly. Used by most rigs.
  • True Fullscreen — own Mission Control Space, smoother for single-output venues. Has its own Space; swipe with 4-finger trackpad.

State badges (per row in modal)

  • Unassigned — placeholder, no display picked
  • Assigned — output picked but role toggle is off (window not open)
  • Live — role toggle on, window painting on the assigned display
  • Identifying — transient (~3.5s) when you click Identify
  • Test Pattern — transient (~8s) when running Color Bars / Focus / Greyscale / Solid

Screens menu (macOS menu bar)

  • Configure Screens (⌘⇧S)
  • Identify All Displays — pops DISPLAY N overlays on every connected display
  • Test Pattern submenu — Color Bars / Focus / Greyscale / Solid
  • Clear All Overlays (Esc)

6. Detection (lyric + Bible)

ACE uses local Whisper (CoreML on Apple Silicon, ~200 ms latency) to transcribe. Then matches transcripts to your imported library via FAISS vector search.

Modes

  • Auto — detects either lyrics or Bible references; switches contextually.
  • Song — locks to lyric matching (best during worship sets).
  • Bible — locks to scripture references (best during sermon).

Sermon recording

Bottom-right Sermon Notes panel:

  1. Optional: title + speaker
  2. Start Recording — backend captures full transcript + extracts key points + auto-detects Bible references
  3. Stop Recording — Export button appears. Export = full transcript + key points + scripture, with the live-transcript snippet that triggered each scripture match shown beneath the canonical verse (e.g. “↳ heard: ‘God works all things for good’” under Romans 8:28)
  4. Past sermons listed under Settings → Data → Sermon History (export anytime)

Smart Bible auto-advance (v1.5.3)

When a Bible verse is currently displayed, ACE listens for the next verse’s text in the live transcript. If the preacher reads verse N+1 directly without saying “next verse,” ACE auto-advances when the cosine-similarity match against verse N+1’s text crosses ≥0.70. This solves the gap where verbal-cue detection (“turn to”) didn’t fire and the screen sat on verse N while the preacher was already on N+1.

Detection Reset (v1.5.3)

⌘⇧R clears stuck vote accumulators, transcript history, and rebuilds the Whisper context-prompt — without stopping audio capture. Replaces the prior workaround of “navigate between Settings tabs to clear state.”

Bible verse lock — ⌘⇧L (v1.5.5)

When ACE catches the right verse but keeps swapping it for AI alternatives as the preacher continues, press ⌘⇧L to lock the current verse. While locked:

  • The verse on screen stays put
  • AI matches still appear in the transcript feed as suggestions
  • Nothing auto-pushes to PROGRAM
  • Manual PageDown / PageUp still advance / retreat freely

Press ⌘⇧L again to unlock. Backend exposes /api/bible/lock, /unlock, /lock/toggle (POST) and /lock (GET) for UI integrations.

What v1.4 fixed

  • Section-stuck bug — detection used to lock onto verse 1 / chorus and refuse to advance even when the band moved on. Whisper context-prompt was self-reinforcing. v1.4 rebalances every section equally; transitions flow with the band.
  • Live transcript not following — same root cause as section-stuck. Fixed by the same change.
  • Slide-doesn’t-reach-HDMI (v1.4.1 patch) — imported slide images now actually paint on audience output, gated on the active Look’s media layer.

7. Stage Layouts

The operator confidence monitor is fully designable.

Stage tab in the toolbar. Pick a preset (current-next, current-notes, current-timers, system-clock, blank) or + New Layout.

Each object: - Type: text (static label), linked_text (live source), shape (box), image, media_placeholder - Source (for linked_text): current_slide, next_slide, system_clock, timer, presentation_name, stage_message, video_countdown - Position / size: drag on canvas (1920×1080 logical) - Style (v1.4 Inspector dropdown): pick a semantic token (stage.text.primary, stage.text.secondary, stage.text.next, stage.text.timer, stage.text.clock, stage.text.muted) or Custom hex

When a token is picked, the active theme drives the rendered color. Custom hex overrides per-object.

Mirror Program toggle (v1.5.3)

Each stage layout now has a Mirror Program checkbox in the editor topbar. When on, the stage screen mirrors whatever is on the audience output (slide image, scripture, song background, video) as a full-bleed backdrop, with the layout’s text objects (current/next lyric, clock, timer, stage message) rendered on top. Speakers see exactly what the audience sees + their confidence cues without turning around.

The mirror reads from the global display theme (display_*) so it always tracks the live audience output, regardless of any per-screen Look override the stage might have.

8. Service Plan

Service Plan tab in the toolbar. Drag songs, scripture, prayer items, announcements, sermon, media into a vertical timeline. Go On Air advances through items.

  • Adding the first item (v1.5.5) — on a fresh install with no service plan, click + Add Item and ACE auto-creates “Untitled Service” dated today, then attaches your chosen item to it. Plan name + date stay editable inline.
  • The standalone + New button was removed in v1.5.6 — the auto-create path above is now the canonical way to start a plan.
  • Planning Center Online sync available if PCO is configured (via direct URL /settings?tab=integrations for power users; future v2 surfaces this in main UI)
  • Each item: assigned theme, layer config, timer, auto-advance rules

9. Quick Screens (⌘J)

Floating panel — bottom-right of the dashboard. Type a message, pick a theme, push to PROGRAM in one click.

  • ⌘J — toggle open/closed (skipped while focused inside an input/textarea so you can still type “j”)
  • ⌘↵ (Cmd+Return) — display
  • Last 10 entries kept on the Recent shelf for one-click reuse
  • Use cases: announcements, prayer calls, offering reminders, lower-thirds

10. Settings highlights

Sidebar groups:

  • General — App, Updates, Audio, Network, CCLI (your church license number — shown on slide footers), Display Hold (how long matches stay on screen)
  • Detection — AI pipeline, Bible parser, Voice Control
  • Output — Screens & Display (includes the “Unified render pipeline” advanced toggle), NDI, MIDI. (ProPresenter + ATEM external-output sub-tabs were hidden in v1.5.7 ahead of the v1.6 ProPresenter Migrator.   • Data — Song Index, Import (now includes “Import from ProPresenter” (v1.6)), PowerPoint rendering

Power users / support reach the integrations panel via direct URL /settings?tab=integrations. v2.0 will introduce a license gateway that proxies free-tier services and gates paid-tier ones behind subscription — no keys, ever.

11. Keyboard shortcuts (full reference)

The canonical shortcut catalog lives in src/lib/shortcuts.ts — this section mirrors it. The shortcut engine already supports per-shortcut customization via localStorage key ws-shortcuts-v1 (functions: saveCustomBinding, resetCustomBinding, resetAllBindings, getEffectiveBindings). The in-app rebinding UI is queued for v1.5.4 — until then, custom bindings need to be set programmatically via the browser DevTools console, e.g.:

// Rebind "Reset Detection" from ⌘⇧R to ⌘⇧Y
const cur = JSON.parse(localStorage.getItem("ws-shortcuts-v1") || "{}");
cur["detection-reset"] = { key: "y", meta: true, shift: true };
localStorage.setItem("ws-shortcuts-v1", JSON.stringify(cur));
window.dispatchEvent(new CustomEvent("ws-shortcuts-updated"));

To restore one shortcut to default: delete the key. To reset every shortcut: localStorage.removeItem("ws-shortcuts-v1").

12. Common workflows

Sunday-morning worship service

  1. Open ACE 30 min before service.
  2. Service Plan tab → drag today’s songs in order (or sync from Planning Center).
  3. ⌘⇧S → confirm Audience + Stage toggles are on, displays assigned, both Live.
  4. Pick the active Look that matches the service mood (e.g. “Standard” / “Worship” / “Scripture”).
  5. Start detection. Walk to the booth, hand over to operator.
  6. During worship: ACE auto-advances through detected songs. Operator presses F1 between songs to clear the screen, or just lets the Display Hold expire.
  7. Sermon: switch to Bible mode (⌃B) or stay on Auto. Sermon Notes start recording; scripture detected and pushed automatically.
  8. After service: stop detection, Export Last Sermon for archiving.

Mid-week conference / lecture

  1. Import the speaker’s deck (PPTX / Keynote / PDF) via the wizard.
  2. Bible references in slide titles + notes are auto-tagged.
  3. During the talk: slides advance automatically as the speaker progresses. → for manual override.

Theater / live show

  1. Pre-load cue order in Service Plan.
  2. Use Quick Screens (⌘J) for ad-hoc announcements between scenes.
  3. Lower-third overlays via props layer (foreground media in Media Bin).

13. Troubleshooting

“I’m on an Intel Mac and ACE never opened / showed a blank window” (fixed in v1.6.0)

Every Intel ACE DMG from v1.0 through v1.5.7 silently shipped with the wrong helper binary inside — Intel installs got an Apple-Silicon backend, and the visible app launched but couldn’t actually transcribe / detect / display. v1.6.0 ships per-architecture helpers (the Intel build now contains the Intel backend, the Apple-Silicon build contains the arm64 backend). If you tried earlier and gave up, grab v1.6.0 fresh from the download button — the install is otherwise the same. See the Intel notice page for the longer explanation.

“App vanished from the dock when the audience window opened” (fixed in v1.5.7)

Single-display Macs only. The audience + stage windows used a window level that outranked macOS’s dock and menu-bar, which on a multi-display rig is correct (audience is on a separate display) but on a single-display rig pushed the operator’s own dock visually out. v1.5.7 detects display count: single-display rigs use a lower window level so the dock stays visible.

“Settings buttons during onboarding (mic / screen recording) did nothing” (fixed in v1.5.5)

The onboarding wizard’s “Open Settings” buttons jump you to the right pane in macOS System Settings via x-apple.systempreferences: deep-links. v1.4 onwards, the IPC handler only allowed http(s):// URLs and rejected those deep-links silently. v1.5.5 extends the allowlist to x-apple.systempreferences: + mailto:.

“F2 / Clear Slide left the slide image stuck on the audience screen” (fixed in v1.5.4)

The keyboard shortcut + the /api/display/clear/{layer} endpoint cleared the lyric overlay but never wiped the slide image — so audience output stayed showing the last slide while subsequent pushes appeared “stuck.” v1.5.4 makes every clear path that targets the slide layer also wipe display_bg_image_path. Right-panel “Clear” worked already because it restored a frontend snapshot; the keyboard / API path now matches.

“+ New button on the Service Plan does nothing” (removed in v1.5.6)

The button used a native window.prompt() which Electron disables since v19, so the click silently no-op’d. As of v1.5.6 the button is gone — instead, hit + Add Item on a fresh service plan and ACE auto-creates “Untitled Service” dated today (v1.5.5). Plan name + date stay editable inline.

“Mic level low” (visualizer is amber)

  • Check the input device in Settings → General → Audio
  • If using an external interface: confirm the interface routes the worship-room mic to ACE’s input
  • Move closer to the mic source if possible

“Detection isn’t picking my song”

  • First check it’s imported: Settings → Data → Song Index
  • Try Manual Search (⌘F) — confirms FAISS has the lyrics indexed
  • If your song is genuinely obscure, ACE falls back to ACR audio fingerprinting (if you’ve configured an ACR key — power user)

“Slide doesn’t reach the audience output”

  • Confirm Audience output is Live (green badge in ⌘⇧S)
  • If active Look has media layer disabled per-screen, slides hide. Check Looks panel.

“Stage output shows wrong layout”

  • Stage tab → confirm the right preset is active
  • Or /api/stage/layouts/active returns the wrong id — check Settings → Output → Stage

“Tour isn’t showing again”

  • Settings → General → “Show tour again” (re-triggers useTourState)

“Update prompt says I’m on v1.2.1 but I just installed v1.4”

  • Fixed in v1.4.0 build pipeline. If you still see this: confirm you replaced /Applications/ACE.app (drag-and-drop the new DMG, accept the replace prompt). The first-launch read of _VERSION.txt from the bundle should report 1.4.x.

“Detection got stuck on verse 1 / chorus and won’t advance”

  • This was the v1.3 bug fixed in v1.4. Confirm Settings → Updates shows v1.4 or later. If yes and it still happens: please report.

“Detection is hallucinating / pushing the wrong verse”

  • Press ⌘⇧R to clear stuck votes + transcript history + rebuild the Whisper context-prompt without stopping audio capture. (v1.5.3)
  • If it persists: switch detection mode (Auto/Song/Bible) — the mode-switch handler runs the same reset automatically.

“PPTX imports as just text on a black background, no slide design”

  • LibreOffice headless conversion failed. Most common causes: profile lock (another LO instance open), missing JRE, or sandbox permission denial.
  • v1.5.3 isolates each import to its own profile dir + adds --nolockcheck, killing the most common case. If it still happens, the backend log now prints the captured stdout/stderr from soffice — paste that to support.

“Mirror Program toggle saved but stage screen didn’t change”

  • Fixed in v1.5.3. If you’re on v1.5.2 or earlier, the workaround is: switch to the Blank layout, save, switch back, save. v1.5.3 makes Mirror apply on the first save.

“Old slide keeps showing after I deleted the PPTX”

  • Fixed in v1.5.3 — delete now clears display_bg_image_path if it was pointing at one of the removed slides, plus the audience window cache-busts on path change. If you’re on an earlier version: F2 (Clear Slide) or ⌘⇧B (Black) clears the stuck image manually.

“Imported the same-named PPTX twice and the audience screen shows the old version”

  • Fixed in v1.5.3 (per-path cache-bust on slide URL). On earlier versions: hard-reload the audience window (Cmd-Shift-R inside the audience BrowserWindow if accessible) or restart the app.

16. Glossary

  • Look — a named bundle of theme + layer toggles + per-screen overrides + transition. The v1.4 unified-render headline.
  • Theme — typography + colors + background style (gradient / solid / image). Standalone, can be referenced by Looks.
  • Layer — one of slide (lyrics), media (background), messages (audience-bottom strip), announcements (banner), props (foreground overlay), bible (scripture).
  • Screen vs Output — Screen = what ACE renders (logical). Output = where it goes (HDMI / NDI / SDI). v2 splits these formally; v1.4 still treats them together.
  • PROGRAM — what’s currently live on the audience output.
  • PREVIEW — what’s queued to go live next (operator stages it, then takes it).
  • Service Plan — the day’s order of items (songs, scripture, prayer, announcements, sermon).
  • Look · Make Live — the action that activates a Look as the current rendering configuration. PROGRAM repaints within ~1 second.
  • Borderless Fill vs True Fullscreen — Borderless fills the display without taking a Mission Control Space; True Fullscreen gets its own Space (swipe between desktops to reach the operator UI).

18. Complete Settings Reference

⌘, to open. Sidebar groups every setting under a high-level category.

General

  • App — app name, theme (light/dark — currently dark only), language
  • Updates — running version, latest release, update notes
  • Audio — input device, sample rate, gain, Recovery (Re-download speech model, Re-run setup wizard) (v1.5.2)
  • Network — backend port, allowed origins, CORS
  • CCLI — your church CCLI license number; renders on slide footer for legal compliance
  • Display Hold — display_hold_strong (how long a strong-confidence match stays on screen), display_hold_weak (weak match)
  • Tour — re-trigger the in-app tour

Detection

  • Detection mode — Auto / Song-only / Bible-only (also on toolbar)
  • Confidence threshold — global AI confidence floor
  • Bible parser — strict / lenient match modes; verse-context window (verses before/after)
  • Voice control — voice-command toggle (e.g. “ACE next slide”)
  • Whisper model — tiny / base / small / medium / large-v3 (memory ↔︎ accuracy trade-off)
  • Transcription language — Auto-detect (bilingual) / English / Spanish / Portuguese / etc.
  • Online transcription — Deepgram on/off (requires API key)
  • Auto-display Bible — toggle whether bible_detection auto-pushes to Program
  • Bible auto-display floor — confidence ≥ 0.60 threshold (v1.5.3, was 0.55)

Output

  • Screens & Display — ⌘⇧S Configure Screens; per-output assignment, role (audience/stage/multi-view), display mode (Borderless Fill / True Fullscreen)
  • Theme background — bg_mode (solid/gradient/image), bg_color, bg_gradient_*, bg_image_path, bg_image_opacity
  • Theme typography — font_family, font_size, font_weight, font_color, subtitle_color, position, h_margin, v_margin, line_reveal, reveal_speed, visible_lines, text_shadow + shadow color/offset
  • Unified render pipeline (v1.4 advanced toggle) — opt-in flag for Look-driven rendering
  • NDI — NDI output toggle, source name, channel
  • MIDI — incoming MIDI shortcut bindings (note-on / CC mappings)

External-output integrations hidden as of v1.5.7 — ProPresenter (OSC bridge) and ATEM (Blackmagic switcher control) sub-tabs were removed from Settings → Output ahead of the v1.6 ProPresenter Migrator.
